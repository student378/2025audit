교육부 감사 PDF 데이터 수집 및 구조화

Claude에게 "Chart.js로 연도별·카테고리별 감사 지적 현황을 시각화하는 HTML을 만들어줘. 필터와 검색도 포함해서."라고 요청하여 초안을 생성합니다.
📝

"아래 JSON 데이터를 기반으로 Chart.js 바차트(연도별), 파이차트(카테고리별), 검색 필터를 갖춘 감사 대시보드 HTML을 만들어줘. 다크 테마, 반응형."

Gemini API AI 챗봇 연동
Google AI Studio에서 Gemini API 키를 발급받고 GAS 프록시를 통해 연동합니다. 사용자가 자연어로 질문하면 994건 데이터 중 관련 사례를 즉시 매칭합니다.
🔑

⚠️
보안: API 키는 반드시 GAS(서버사이드)에만 저장. HTML에 직접 넣으면 안 됩니다.

챗봇이 하는 일

자연어로 질문하면 994건 감사 데이터 중 관련 사례를 즉시 찾아줍니다. 기존 키워드 검색으로는 놓치는 의미적 유사 사례까지 매칭합니다.

student378.github.io/2025audit/ — 💬 AI 검색챗봇
Q. 시설 공사 관련해서 어떤 유형의 지적이 많이 발생했어?
시설·공사 관련 지적은 118건으로 전체 12%를 차지합니다. 주요 유형은 ① 수의계약 근거 미비(43건) ② 설계·감리 동일업체 발주(28건) ③ 안전점검 기록 미보존(18건) 순입니다. 특히 2천만원 이상 수의계약 시 법령 근거를 구체적으로 기재하지 않은 사례가 가장 많습니다.
Q. 우리가 올해 특히 주의해야 할 항목은?
최근 3년 기준 증가 추세인 지적 유형은 ① 개인정보 처리방침 미갱신 ② 연구비 종료 후 집행 ③ 근로장학금 실근무 기록 미비입니다. 이 세 항목을 우선 자체점검하시길 권장합니다.
효과적인 질문 예시
상황추천 질문
감사 전 준비"우리 학교 학사팀이 감사 전 가장 먼저 확인해야 할 사항은?"
유사 사례 검색"장학금 지급 관련 지적 사례를 모두 보여줘"
트렌드 파악"최근 3년간 새로 증가하는 지적 유형은 무엇이야?"
조치 방법"법인카드 심야 사용이 지적됐을 때 어떻게 해명했어?"
위험도 확인"시설팀 기준 고위험 항목만 요약해줘"
⚠️
AI 챗봇은 데이터 기반 참고 자료입니다. 최종 감사 대응은 반드시 관련 법령과 담당자 판단을 병행하세요.

자체점검 시스템 연계 활용
감사 사례 → 체크리스트 → 사전 예방의 선순환
연계 흐름 (4단계)
1
감사대시보드 "담당업무별 가이드" 탭 확인
우리 부서와 관련된 고위험 항목을 확인합니다. 각 항목 옆의 ✅ 자체점검 §X-X 배지를 클릭하면 해당 자체점검 영역으로 바로 이동합니다.
2
자체점검 입력화면에서 해당 항목 점검
student378.github.io/check/ 접속 → 부서 선택 → 해당 영역의 세부 항목 이상유무 입력
💡
점검 입력 중 "🔍 감사 사례 참고" 버튼을 누르면 대시보드를 바로 열 수 있습니다.
3
이상있음 항목 → AI 챗봇으로 유사 사례 확인
이상있음으로 표시된 항목이 있다면, AI 챗봇에 "우리 학교에서 [해당 항목] 관련 지적 사례와 조치 방법을 알려줘"라고 질문하여 사전 대응 방안을 준비합니다.
4
부서장 확인 후 감사실 제출
모든 항목 점검 완료 후 부서장 서명 → 감사실 제출. 감사실 관리자 현황판에서 실시간으로 제출 현황을 확인합니다.

실전 활용 시나리오
상황별 사용 방법을 단계별로 안내합니다
📅 시나리오 1 — 연간 감사 준비 (감사 3개월 전)
  • 감사대시보드 "종합 대시보드" 탭에서 최근 3년 트렌드 파악
  • AI 챗봇에 "올해 중점 감사 예상 항목을 알려줘" 질문
  • "담당업무별 가이드" 탭에서 우리 부서 고위험 항목 목록 출력
  • 자체점검 시스템으로 전 부서 사전 점검 실시
  • 감사실에서 전 부서 제출 현황 실시간 확인
🔍 시나리오 2 — 특정 사안 발생 시 즉시 검색
예: 시설팀에서 2,500만원 수의계약을 체결해야 할 상황
  • AI 챗봇에 "2천만원 이상 수의계약 관련 지적 사례와 요구 서류 알려줘" 질문
  • "지적사례 목록" 탭에서 "수의계약" 키워드로 유사 사례 전체 검색
  • 지적 사례의 조치 내용을 참고하여 서류 사전 준비
  • 자체점검 §4-1(계약관리) 항목 점검 후 이상유무 기록
📊 시나리오 3 — 신임 담당자 온보딩
새로 부임한 담당자가 우리 부서 감사 위험 요소를 빠르게 파악해야 할 때
  • "담당업무별 가이드" 탭에서 담당 부서 선택
  • 고위험 항목부터 순서대로 실제 현황과 대조
  • AI 챗봇에 "학생지원팀 신임 담당자가 가장 먼저 확인해야 할 것은?" 질문
  • "수감자료 준비가이드" 탭에서 평소 비치해야 할 서류 목록 파악
✅
